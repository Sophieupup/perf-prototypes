业务规则提示 · HCM 区域为什么拆 2 列
HCM 区域（签单）：用于 2.3 签单明细 + 4.6 签单/签单毛利目标的归集 · 通常 = 谈下这单的 MU 所在区
HCM 区域（收入）：用于 2.4 回款明细 + 4.6 回款收入/回款毛利目标的归集 · 通常 = 实际交付/服务团队所在区
默认：上传后系统自动把"签单"复制到"收入"两列同值。
例外修正：跨区项目（如签单 = 央企 / 交付 = 华东）admin 可在列表手动改"收入"列，琥珀高亮 ✎ 标识。
初始导入：admin 用「下载模板」拿到空白 Excel，填好后上传 → 全量初始化。
后续新签合同：系统识别后弹窗只问 5 个字段（合同编号 / 项目名称 / MU / HCM 区域签单 / HCM 区域收入），admin 一次性补完后 append。
